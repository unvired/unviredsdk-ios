  • IDataStructure

    IDataStructure represents application data as a business object.

    Unvired data model consists of a set of Business Entities.
    Business Entity is a logical construct consisting of a Header Item and other children Items (Note: Structure and Item are used interchangeably). Business Entity can have only one Header and many children Items.

    1. Header / Item is a collection of Fields.
    2. Field is a field name and value pair.

    A sample data model is provided below:

      <Application name="CRM" description="CRM Mobile App" version="1.0" dbversion="1.0" package="com.mycompany.crm" namespace="mycompany">
  <BusinessEntity name="CUSTOMER" description="Customer">
      <header className="com.indience.template.saperp.so.be.CUSTOMER" description="" name="CUSTOMER_HEADER">
          <Field name="CUSTOMER_NO" description="Customer No" sqlType="TEXT" isGid="true" length="10" mandatory="true" />
          <Field name="CUSTOMER_NAME" description="Customer Name" sqlType="TEXT" length="12" mandatory="true" />
          <Field name="STREET" description="Street" sqlType="TEXT" length="50" mandatory="false" />
          <Field name="CITY" description="City" sqlType="TEXT" length="50" mandatory="false" />
          <Field name="PIN_CODE" description="Pin Code" sqlType="TEXT" length="10" mandatory="false" />
      </header>
      <item className="com.indience.template.saperp.so.be.CUSTOMER_SHIP_TO_ITEM" description="" name="CUSTOMER_SHIP_TO_ITEM">
          <Field name="CUSTOMER_NO" description="Customer No" sqlType="TEXT" isGid="true" length="10" mandatory="true" />
          <Field name="CUSTOMER_NAME" description="Customer Name" sqlType="TEXT" length="12" mandatory="true" />
          <Field name="STREET" description="Street" sqlType="TEXT" length="50" mandatory="false" />
          <Field name="CITY" description="City" sqlType="TEXT" length="50" mandatory="false" />
          <Field name="PIN_CODE" description="Pin Code" sqlType="TEXT" length="10" mandatory="false" />

  </BusinessEntity>
